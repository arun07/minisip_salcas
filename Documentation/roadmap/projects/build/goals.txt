Introduction
------------

This document presents the long-term goals for the Build Working Group.  
These serve to guide the group on a clear path for fulfilling the group'
s specific functions within The Project.

===========
Group Goals
-----------

1) "The system must be built before it can be used."
  a)  The build system must remain stable, even through frenzied changes.

2) "Test suites can validate designs, prevent regressions, and more."
  a) Every library and application must have a complete test suite.
  b) Each public interface (API header) must have a unit test covering it.
  c) Additional test services must be available.

3) "Software gains tremendous value when it gets distributed for use."
  a) Every package should pass their 'distcheck' tests.
  b) Package contents must be reviewed before being distributed.
 
4) "Distrubtion requires more than simply tossing tarballs on a website.
  a) The Build Group oversees the Ports, Sites, and Docs groups through 
     the release process.
